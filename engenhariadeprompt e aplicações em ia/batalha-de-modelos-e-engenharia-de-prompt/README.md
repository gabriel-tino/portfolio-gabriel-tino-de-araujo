# # Batalha de Modelos: Engenharia de Prompt (XML)

Este repositório registra os dados e análises da atividade de desenvolvimento Front-end utilizando prompts estruturados para testar a fidelidade e precisão de diversos LLMs[cite: 3].

## ## 🎯 Objetivos da Atividade
*   **Análise Crítica:** Avaliar a precisão técnica e a conformidade dos modelos a instruções estritas[cite: 3].
*   **Metacognição:** Identificar forças e limitações de cada arquitetura para tarefas de programação[cite: 3].
*   **Fidelidade ao XML:** Testar se a IA respeita tags, cores e diretrizes técnicas sem ignorar restrições[cite: 3].

## ## 🛠️ Protocolo de Execução
A tarefa consiste em enviar um **Prompt Estruturado em XML** para gerar uma página HTML *Single Page* com CSS integrado[cite: 3].

### ### Modelos Testados
*   ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca[cite: 3].

### ### Estrutura do Prompt (Tags XML)
*   `<objetivo>`: Criar página HTML5 única com CSS interno[cite: 3].
*   `<diretrizes_design>`: Layout responsivo, paleta de cores e tipografia específica[cite: 3].
*   `<obrigatoriedades_tecnicas>`: Menu funcional, seção de portfólio e rodapé[cite: 3].
*   `<metrica_obrigatoria>`: Estimativa de tokens gerados ao final da resposta[cite: 3].

## ## 🧠 Reflexão Crítica
O exercício propõe a identificação de qual ferramenta é ideal para cada cenário[cite: 3]:
1.  **Prototipagem Rápida:** Qual modelo obedece melhor à estrutura básica[cite: 3]?
2.  **Códigos Complexos:** Qual IA mantém a integridade lógica em estruturas extensas[cite: 3]?
3.  **Verbosidade:** Análise da diferença no consumo de tokens para o mesmo prompt[cite: 3].

> **Nota de Avaliação:** A nota foca na profundidade da análise técnica e na obediência do modelo às restrições, independentemente da estética gerada[cite: 3].
